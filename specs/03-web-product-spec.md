# Web Product Spec

## 文档状态

- Status: Active
- Scope: Web 单页产品的用户功能与体验要求

## 目标

Web 是当前最成熟的产品入口，必须提供完整的本地闭环体验。

## 已实现能力

状态：`Implemented`

- 多条 2FA 条目管理
- 输入 Base32 密钥并生成验证码
- 倒计时与自动刷新
- 条目备注名
- 本地持久化
- 扫码导入
- 图片二维码导入
- 多选分享
- `otpauth://...` 原始列表分享
- 基于当前域名的 `?import=BASE64CONTENT` 链接导入
- 点击复制验证码
- GitHub 仓库入口
- Self Provider 同步面板

## 页面职责

### 主页面

负责：

- 展示当前 Vault 条目
- 展示倒计时与验证码有效窗口
- 支持新增、编辑、删除条目
- 支持多选与批量分享

### 导入入口

负责：

- 从文本导入 `otpauth://...`
- 从相机扫码导入
- 从图片识别二维码导入
- 从分享链接 `?import=...` 恢复条目

### 分享对话框

负责：

- 展示原始 `otpauth` 列表
- 提供复制原始内容能力
- 生成当前域名分享链接
- 为未来安全分享模式保留入口

### Self Provider 卡片

负责：

- 输入远端地址
- 输入用户主密钥
- 建立会话
- 查看同步状态
- 主动拉取 / 推送

## 体验要求

### 验证码显示

- 默认显示连续数字，中间不插空格
- 必须支持一键复制
- 必须清晰显示剩余有效秒数
- 刷新节奏应稳定，不抖动

### 导入体验

- 无效密钥必须给出明确错误提示
- 导入结果要区分：成功、跳过、失败
- 同一密钥重复导入时应避免无意义重复写入

### 数据安全

- 本地模式下不依赖服务端
- 本地持久化时要有版本迁移能力
- 导入分享链接后应尽量避免污染 URL 历史

## 当前实现短板

状态：`In Progress`

- Web UI 已具备功能闭环，但后续仍需继续组件化与状态拆分
- Sync Provider 抽象已存在方向，但 UI 仍以 Self Provider 为主
- 更复杂的安全分享模式仍需进一步定义

## 后续规划

状态：`Planned`

- 更明确的 Vault / Provider 管理 UI
- 更强的条目搜索、分组与标签体系
- 更完整的冲突提示与同步历史展示
